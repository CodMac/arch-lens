# Java 元素全限定名 (QN) 提取约束规约

本文件定义了 `go-treesitter-dependency-analyzer` 在解析 Java 源码时生成 `QualifiedName` 的标准格式与约束，用于确保符号消解和依赖分析的一致性。

## 1. 基础原则
- **分隔符**: 统一使用英文句点 `.` 作为层级分隔符（涵盖包名、类名、成员名、局部作用域）。
- **层级结构**: `[Package].[TopLevelClass]$[InnerClass].[Member]([Params])$[LocalScope]`。
- **唯一性**: QN 必须在全局上下文中唯一。对于无自然名称或同名但不同实体的元素（如匿名类、局部变量），通过序号后缀 `$n` 进行区分。

## 2. 核心元素提取规则

### 2.1 包 (Package)
- **格式**: `packageName`
- **来源**: 提取自 `package_declaration` 节点。
- **示例**: `com.example.service`

### 2.2 类型声明 (Class, Interface, Enum, Annotation)
- **格式**: `[ParentQN].[ClassName]`
- **说明**: 
    - 顶级类直接挂在包名下。
    - 内部类挂在父类 QN 下。
- **示例**: `com.example.User` (顶级类), `com.example.Outer.Inner` (内部类)

### 2.3 方法 (Method & Constructor)
- **格式**: `[ParentQN].[MethodName]([ParamType1],[ParamType2],...)`
- **重载处理**: 为了区分重载方法，QN 必须包含参数类型列表（不含参数名）。
- **构造函数**: 方法名使用类名，并带上参数列表。
- **泛型擦除**: QN 中的参数类型仅保留原始类型，忽略泛型符号（如 `List<String>` 提取为 `List`）。
- **示例**: 
    - `com.example.User.setName(String)`
    - `com.example.User.xxx(com.example.Student, String)` (参数类型保留代码的原始类型)
    - `com.example.User.User(String,int)` (构造函数)
    - `com.example.User.save()` (无参方法)

### 2.4 字段 (Field) & 枚举常量 (Enum Constant)
- **格式**: `[ParentQN].[FieldName]`
- **示例**: `com.example.User.id`, `com.example.Status.ACTIVE`

### 2.5 变量 (Variable - 局部变量 & 参数)
- **格式**: `[ParentQN].[VariableName]`
- **作用域修正**: 局部变量的 `ParentQN` 可能是方法、Lambda 表达式或一个逻辑块 (`block$n`)。
- **示例**: `com.example.User.save(String).isSuccess`

## 3. 特殊容器与匿名元素

对于没有自然名称的元素，系统通过 **硬编码前缀 + 出现序号** 的方式生成标识符。

| 元素类型 | QN 标识符格式 | 说明 |
| :--- | :--- | :--- |
| **匿名类** | `anonymousClass$n` | 如 `new Runnable() { ... }` |
| **Lambda** | `lambda$n` | `(x) -> x + 1` |
| **逻辑块** | `block$n` | 方法内的 `{ ... }` 块或 `if/for` 的分支块 |
| **静态块** | `$static$n` | `static { ... }` |
| **实例块** | `$instance$n` | 类中的 `{ ... }` 初始化块 |
| **方法引用** | `method_ref$n` | `System.out::println` |

**注**: 序号 `$n` 基于该元素在父级作用域内出现的顺序，从 1 开始。

## 4. 符号冲突与重复处理

在同一作用域内，如果出现同名的非匿名元素，系统会追加递增序号以保证 QN 唯一：
- 第 1 个出现: `name`
- 第 2 个出现: `name$2`
- 第 n 个出现: `name$n`

## 5. 语法糖增强 (Syntactic Sugar)

系统会为 Java 隐式生成的代码提取对应的 QN，标识为 `IsFormSugar: true`：
- **默认构造函数**: 如果类中未显式定义构造函数，会自动生成 `ClassName()`。
- **Enum 辅助方法**: 每个枚举类会自动提取 `values()` 和 `valueOf(String)`。
- **Record 组件**: 为 Java Record 自动提取同名字段及对应的 Getter 方法（如 `name()`）。
